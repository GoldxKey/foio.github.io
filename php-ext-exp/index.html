PHP扩展问题定位经验小结

October 24, 2014

Reading time ~1 minute

使用debug版的PHP

编译PHP时加上参数：

./configure  --enable-debug

使用core文件

取消core文件大小限制

ulimit -c unlimited

通过修改/proc/sys/kernel/core_pattern可以控制core文件保存位置和文件格式

echo "/tmp/corefile/core-%e-%p-%t" > /proc/sys/kernel/core_pattern

使用corefile定位扩展中的问题

gdb php -c corefile //线上出错的文件和行号
bt //显示错误处的函数调用栈

在程序中打印日志

可以使用php_printf(用法同c语言的printf)打印原始变量的值。

使用PHP内置server

PHP5.4之后内置了server，极大的方便了扩展的开发。

$ php -S localhost:8000   //在当前文件夹下启动server，默认会寻找index.php
